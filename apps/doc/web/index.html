Status

This ADL Workbench 1.5 beta build supports ADL 1.5 templates, external references and significantly improved validation. The key features of ADL/AOM 1.5 are supported, and implementation of the remaining features will occur in the series of subsequent beta releases following this beta release in the near future. For a list of what is implemented already versus what remains, see the Learning ADL 1.5 page. The longer term future of this tool will be determined by a joint working programme of openEHR Foundation and IHTSDO. Release notes .

Archetypes

You need archetypes and optionally templates. Where you can get them:

  • You can write them yourself, or you may already have some in your organisation
  • You can get test archetypes from the new openEHR Subversion repository , which includes:
    • ADL 1.5 validation archetypes and templates
    • openEHR example archetypes and templates
    • some early CEN EN13606 example archetypes
  • The openEHR archetypes are available from the Clinical Knowledge Manager (CKM). Obtain a login in order to download all the archetypes. This repository has over 250 archetypes, in various stages of development by a large number of international experts, and includes:
    • clinical archetypes
    • administrative archetypes
    • demographic archetypes

See here for detailed set up instructions. Archetypes and templates can be put anywhere on your file system. If you are working on some archetypes or templates of your own that are based on existing archetypes, it is useful to put the latter in a 'reference' location, and your own in a 'work' location. You can configure ADL Workbench to know about reference and work locations, and it will visually indicate which archetypes are yours, and which are the reference ones.
