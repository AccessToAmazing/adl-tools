What else do I need?

You need archetypes and optionally templates. Where you can get them:
  • You can write them yourself, or you have have some in your organisation
  • You can get test archetypes from this Subversion repository:  http://www.openehr.org/svn/knowledge2/TRUNK , which includes:
    • ADL 1.5 validation archetypes and templates
    • openEHR example archetypes and templates
    • CEN EN13606 example archetypes
  • The openEHR archetypes are available from the Clinical Knowledge Manager (CKM) at http://www.openEHR.org/knowledge. Obtain a login in order to download all the archetypes. This repository has over 250 archetypes, in various stages of development by a large number of international experts, and includes:
    • clinical archetypes
    • administrative archetypes
    • demographic archetypes
Archetypes and templates can be put anywhere on your file system. If you are working on some archetypes/templates of your own that are based on existing archetypes, it is useful to put the latter in a 'reference' location, and the former in a 'work' location. You can configure the tool to know about reference and work locations, and it will visually indicate which archetypes are yours, which are the reference ones.

Overview

The ADL Workbench is a visual front-end for the openEHR reference ADL parser. It can be used for compiling, viewing and batch testing archetypes created for any reference model, and in any number of languages. The tool is based on the ADL 1.5 drafts of the Archetype Definition Language (ADL) specification, the Archetype Object Model (AOM) specification, and the openEHR archetype Profile specification.

The two parts of the tool are the Compiler and the Tester, which are available from the tabs on the right-hand side of the application.
