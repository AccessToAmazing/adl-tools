ADL Workbench (AWB) - Tool Configuration

Download and Help pages

Set-up

In order to work with the examples described below, the following configuration is advised. Experienced users may skip this step.

Repository setup

You should download the CKM and the openEHR test archetype repositories. The steps to do this are as follows.

For the Clinical Knowledge Manager (CKM) repository:

  • Login to the repository at http://www.openehr.org/knowledge.
  • Go to Archetypes > Bulk Export and select all archetypes for download; save in any location. For the purposes of this manual, we will refer to the location as $CKM.
  • In the AWB, select Repository > Set Repository
  • On the dialog, use the large '+' button to add a new profile called 'CKM'.
  • Set the Reference repository to $CKM, as described above.
  • Save the new profile.

The openEHR test archetype repository is a Subversion repository located here. If you have a Subversion client such as TortoiseSVN installed on your machine, use it to check out this repository. If you do not have a Subversion tool, you can download the repository as a .zip file from this page. We will refer to the location of this repository on your file system as $knowledge2. Now, there is actually more than one logical group of archetypes in the repository, so the configuration is as follows.

  • In the AWB, select Repository > Set Repository.
  • On the dialog, use the large '+' button to add a new profile called 'ADL 1.5 test' or similar.
  • Set the Reference repository to $knowledge2/archetypes/ADL_1.5_test.
  • Save the new profile.
  • Repeat the above steps and create another new profile called 'openEHR examples' or similar.
  • Set the Reference Repository to $knowledge2/archetypes/openEHR_examples/demographic_template/Reference.
  • Set the Work Repository to $knowledge2/archetypes/openEHR_examples/demographic_template/Working.
  • Save the new profile.

If you are interested in EN 13606, you can use the above method to create a further profile called 'EN13606 examples' or similar, using the Reference Repository path $knowledge2/archetypes/en13606_examples.

RM Schemas

You will need to have all the relevant RM schemas selected for these profiles to work. Assuming you have followed the above instructions, you do this as follows:

  • Select Tools > RM Schemas
  • Ensure that the openehr, adltest, and en13606 (if relevant) schemas are checked, as shown here.
  • Save the configuration.

You can now use the Repository > Set Repository dialog to switch between the profiles at will. The remaining sections below describe how to see the ADL/AOM 1.5 changes using the archetypes you have configured above.

The .cfg file

The result of the above configuration creates entries in the application configuration file, which will be in one of the following places, depending on your operating system:

  • Windows XP/Vista: C:\Documents and Settings\<username>\AppData\Local\openEHR\adl_workbench\adl_workbench.cfg
  • Windows 7: C:\Users\<username>\AppData\Local\openEHR\adl_workbench\adl_workbench.cfg
  • Mac OSX: $HOME/openEHR/adl_workbench/adl_workbench.cfg
  • Linux: $HOME/openEHR/adl_workbench/adl_workbench.cfg

This file is in dADL format, and will typically include lines such as the following, corresponding to the above configuration.

		editor_app_command = <"cmd /q /d /c start \"\" /b">

	>
	profile = <
		current_profile_name = <"adl_test">
